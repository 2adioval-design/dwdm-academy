# DWDM Architect Center

DWDM Architect Center is the production-style LMS foundation for the DWDM Academy program. It turns the six source Word documents into a real course platform with:

- a Next.js learner experience
- course and section data stored in SQLite
- role-based auth for learners, instructors, and admins
- persistent progress, notes, and bookmarks
- Stripe Checkout plumbing for paid enrollments
- instructor and admin back-office views

## Stack

- **Next.js App Router** (v14 — Node 18 compatible)
- **React 18**
- **SQLite** with better-sqlite3
- **Stripe Checkout Sessions** for payments

## Local setup

```bash
# 1. Copy environment file
cp .env.example .env

# 2. Install dependencies
npm install

# 3. Generate catalog from your Word files (place .docx files in ~/Downloads)
python scripts/extract_catalog.py

# 4. Seed the database
npm run db:seed

# 5. Start the dev server
npm run dev
```

Open http://localhost:3000 — you'll be redirected to /login.

## Payments

Stripe is wired for Checkout Sessions and webhook fulfillment. To activate paid enrollment flows:

```env
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
```

Without Stripe credentials the pricing page exposes a **sandbox enrollment path** so the full LMS can be exercised end-to-end at no cost.

## Data storage

The database path is controlled by `DATABASE_FILE`. Default: `./data/dwdm-architect-center.db`

## Reset database

```bash
npm run db:reset
```

This deletes the database file and re-seeds from scratch.

## Deployment

### Vercel (recommended)

```bash
npm i -g vercel
vercel
```

Set environment variables in Vercel dashboard:
- `DATABASE_FILE` → `/tmp/dwdm.db` (or use Turso/PlanetScale for persistence)
- `JWT_SECRET` → a random 64-char string
- `STRIPE_SECRET_KEY` → your Stripe key
- `STRIPE_WEBHOOK_SECRET` → from Stripe dashboard

### Railway

1. Connect GitHub repo in Railway dashboard
2. Set environment variables
3. Railway auto-detects Next.js and deploys

> **Note**: SQLite is ephemeral on serverless platforms. For persistent storage use a hosted SQLite service (Turso) or migrate to PostgreSQL via Prisma.
